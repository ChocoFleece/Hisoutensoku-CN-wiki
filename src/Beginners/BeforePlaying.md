### 确保网络连接稳定和电脑性能有余

::: info 一、检查网络波动情况
按下 Win+R，复制粘贴 `ping baidu.com -t` 并回车，等待一会儿

延迟波动范围上下 15ms 左右都是正常情况，可以接受

**也可以检查游戏对局内正中央下方数字有没有一直在跳变，若有则很可能说明有人是 Wifi，网络不稳定**

![像这样延迟波动不大则可以认为是稳定的，进行游戏不会卡顿](https://bu.dusays.com/2023/08/22/64e4a1d8cbfce.png =500x)

![延迟很不稳定的 Wifi，不建议进行游戏，没有网线网络的话推荐尝试手机流量USB共享给电脑，顺便说一下流量是支持IPV6且可以用于联机的](https://bu.dusays.com/2023/08/22/64e4a347231c8.png =500x)

::: tip 手机流量如何USB共享给电脑
1. **手机关掉 Wifi，打开数据流量，USB连接电脑，电脑也关掉 Wifi，注意提前关掉其他可能会消耗流量偷跑流量的东西（如迅雷、浏览器后台视频等）**

2. ![](https://bu.dusays.com/2023/08/25/64e789346219d.png =250x)

3. ![](https://bu.dusays.com/2023/08/25/64e7893474375.png =250x)

4. ![](https://bu.dusays.com/2023/08/25/64e78933a4154.png =250x)

5. 然后右下角网络图标应该是这样 ![](https://bu.dusays.com/2023/08/25/64e78933460b7.png) ，就可以了
:::

::: info 二、检查电脑性能有没有高占用

**显卡驱动等没装可能会导致卡顿**

常见的占用情况：复制大量文件占用磁盘、解压大文件、扫盘杀毒、导出视频、压制视频、浏览器开太多网页视频，以及笔记本未插电源，电量不足进入省电模式等；
请避免在这些情况下玩游戏，可能会造成加载速度慢，顿卡，掉帧等问题

可在底部任务栏空白处，右键，点击任务管理器，查看资源占用情况

:::

### **使用 Swarm 等中转 IP 在大厅里联机**

::: info 使用说明

**Swarm 通过服务器中转数据包的方式来帮助联机，只需要建主方打开软件即可，连接方无要求**

1. 双击 [ 启动 ] 游戏+Swarm联机中转+大厅IP配置.bat

![](https://bu.dusays.com/2024/01/21/65accd5cb5c9d.png =300x)

2. 点更新服务器列表

3. 点中转然后复制

![swarm和则之间启动的 **先后顺序不影响使用**，注意不要改动10800的数字](https://bu.dusays.com/2023/08/18/64de5a1b6da29.png =200x)

4. 在 ini 配置内粘贴复制来的IP，**按 Ctrl+S 保存**，不需要重开游戏。

![大厅IP配置](https://bu.dusays.com/2024/01/21/65accd2b6f5e2.png =280x)

**配置完毕swarm，就可以自己建主，让对方再进入即可**

::: warning 
**如果你用 swarm-old 遇到别人进入后黑屏又退出的情况，可尝试使用 swarm-new**

:::

::: warning
**提醒注意使用 swarm 不要绕远路了，如果发现延迟离奇的大，确认双方 swarm 延迟列表情况后，选择最优线路即可**

**如果支持 AutoPunch 或者 IPV6 直连，那是最好的**

**下面多图，手动点击放大查看**

![swarm 服务器 - 联机说明](https://bu.dusays.com/2024/01/21/65acce946fa86.webp =100x) ![正确连接1](https://bu.dusays.com/2024/01/21/65accec8ef27a.webp =100x) ![正确连接2](https://bu.dusays.com/2024/01/21/65accedc69106.webp =100x) ![错误连接-绕远路1](https://bu.dusays.com/2024/01/21/65acceed9202e.webp =100x) ![错误连接-绕远路2](https://bu.dusays.com/2024/01/21/65accefb8c88c.webp =100x)

 ::: info 
**服务器所在地区： 124——上海 ；159——广东 ；42——四川 ；152和101——北京**

**实际延迟 ≈ 主机到服务器的距离 + 客机到服务器的距离**

符合上述四个省份的可以无脑选自己所在省份，其他的人一般选用离得近的地区作为中转

如果双方网络延迟过大，很有可能是绕了远路，可以检查联机双方的地区来选择合适的中转服务器

中转一般选“标准模式”，wifi对局可尝试使用“玄学模式”（确实是玄学）
左下角配置可以修改链接端口，默认10800，一般不需要修改，
勾选修复端口地址以允许观战
:::

### 注意正中央网络延迟并调整左下角延迟帧数

::: info 进入对局后

- **如果发现对手一直没反应，请理解并耐心等待，他可能身边有点急事需要离开一会儿；可以查看下QQ群消息和大厅聊天框，也许他正在那边打字**

- **如果发现自己和对手实力悬殊（对手太强或者太新），可以在QQ群或聊天框内交流是否还要继续玩**

- **最好不要无端故意中途ESC结束对局，会导致无法保存Rep，若有紧急情况要离开可以在事后QQ群或者聊天框通知对手**

- **2023.11.30 Hagb制作了SaveRep的mod可以在异常情况下紧急保存rep，虽然ESC也可以保存Rep了但是依然不提倡“节省时间”而在对局时ESC**

- **打完结束后可以按两下X框框两声，并回复谢指教，谢则之类的话感谢对战**


::: tip 注意正中央网络延迟并调整左下角延迟帧数

![giuroll 0.6.3+ 数据说明](https://bu.dusays.com/2024/01/21/65acd056779ab.webp =600x)

![根据正中央的网络延迟调整左下角的输入延迟帧数](https://bu.dusays.com/2024/01/21/65acba0768ae5.webp =700x)

**1P方请参考正下方的延迟，使用 “大键盘加减号(在 Backspace 键的左边)” 调整左下角 delay 值，调整规则可参考下图**

**简单来说就是：从d1开始，若发生频繁回滚现象则增大，直到不会发生回滚为止**

 **giuroll 0.6.0版本开始**，增加“单方回滚”特性：对战双方各自按大键盘加减号（Backspace退格键左边的那个），来调整自己的左下角延迟帧数，互相不影响对方；

::: info
假设游戏内正下方的数字显示 40，说明双方网络延迟是 80ms 左右，就相当于总共延迟 5 帧，此时

**1P 设置左下角 d1，就选择延迟 1F（1P 的指令输入像练习模式一样流畅），但是就需要 4F 的回滚帧（1P 画面里的对手发生回溯与瞬移的情况频发）；**

**2P 设置左下角 d3，就选择延迟 3F（2P 输入的指令过 3F 才会实际输入游戏内），不过只需要 2F 的回滚帧（2P 画面里的对手基本不会发生回溯与瞬移）**
:::
